# 认知陷阱平台 - 交互体验测试方案

## 1. 测试目标

确保所有场景的交互体验流畅、直观，避免提及线性期望等认知偏差，提供良好的用户体验。

## 2. 测试范围

### 2.1 场景覆盖
- 咖啡店线性思维场景
- 恋爱关系场景
- 商业战略推理游戏 (game-001)
- 公共政策制定模拟 (game-002)
- 个人理财决策模拟 (game-003)
- 历史案例场景 (hist-*)
- 高级游戏场景 (adv-game-*)

### 2.2 交互元素测试
- 决策选项选择机制
- 状态显示更新
- 反馈信息展示
- 步骤导航流程
- 错误处理机制

## 3. 测试用例

### 3.1 基础交互测试

#### 3.1.1 场景启动测试
- **测试步骤**：
  1. 从场景列表选择任意场景
  2. 点击"开始挑战"按钮
  3. 观察场景是否正确加载
- **预期结果**：
  - 场景页面正确显示
  - 场景描述和初始状态正确显示
  - 决策控件正确加载

#### 3.1.2 决策选择测试
- **测试步骤**：
  1. 在场景中选择一个决策选项
  2. 观察选项是否被正确选中
  3. 选择另一个选项，观察是否正确切换
- **预期结果**：
  - 选项被高亮显示
  - 提交按钮变为可见/可用
  - 之前选择被取消

#### 3.1.3 决策提交测试
- **测试步骤**：
  1. 选择一个决策选项
  2. 点击"提交决策"按钮
  3. 观察反馈信息
- **预期结果**：
  - 决策被成功提交
  - 状态信息正确更新
  - 反馈信息清晰明了

### 3.2 多步骤场景测试

#### 3.2.1 步骤导航测试
- **测试步骤**：
  1. 启动一个多步骤场景 (如game-001)
  2. 完成第一步决策
  3. 观察是否自动进入下一步
- **预期结果**：
  - 步骤进度条正确更新
  - 新的情境描述显示
  - 新的决策选项可用

#### 3.2.2 步骤完成测试
- **测试步骤**：
  1. 完成多步骤场景的所有步骤
  2. 观察是否显示完成画面
- **预期结果**：
  - 显示场景完成画面
  - 提供决策分析
  - 提供后续操作选项

### 3.3 特殊场景测试

#### 3.3.1 咖啡店场景测试
- **测试步骤**：
  1. 启动咖啡店线性思维场景
  2. 调整员工数量滑块
  3. 调整营销投入滑块
  4. 提交决策
- **预期结果**：
  - 滑块值正确显示
  - 状态数值正确更新
  - 反馈信息不提及线性期望

#### 3.3.2 游戏场景测试
- **测试步骤**：
  1. 启动任一game-*场景
  2. 选择决策选项
  3. 提交决策
  4. 观察反馈
- **预期结果**：
  - 选项选择机制正常
  - 多步骤导航正常
  - 反馈信息中性且有帮助

## 4. 验收标准

### 4.1 交互流畅性
- [ ] 所有场景加载时间 < 3秒
- [ ] 决策选择响应时间 < 0.5秒
- [ ] 页面切换流畅无卡顿

### 4.2 用户体验
- [ ] 界面清晰易懂
- [ ] 操作指引明确
- [ ] 错误提示友好

### 4.3 功能完整性
- [ ] 所有场景可正常启动
- [ ] 所有决策选项可正常选择
- [ ] 所有反馈信息正确显示

### 4.4 内容准确性
- [ ] 不提及线性期望等认知偏差
- [ ] 提供中性且有帮助的反馈
- [ ] 状态更新准确

## 5. 测试执行

### 5.1 手动测试
1. 按照上述测试用例逐一执行
2. 记录测试结果
3. 标记发现的问题

  // 测试决策选择
  const decisionSelection = await testDecisionSelection();
  console.log(`决策选择测试: ${decisionSelection ? '通过' : '失败'}`);
  
  // 测试反馈显示
  const feedbackDisplay = await testFeedbackDisplay();
  console.log(`反馈显示测试: ${feedbackDisplay ? '通过' : '失败'}`);
  
  console.log('交互体验测试完成');
}
