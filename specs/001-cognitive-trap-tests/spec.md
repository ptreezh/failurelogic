## User Scenarios & Testing *(mandatory)*

### User Story 1 - 指数增长误区测试 (Priority: P1)

用户通过选择题形式的测试，体验和学习指数增长误区，特别是对2^200这样的天文数字缺乏直观认识，以及对复合增长效应的低估。

**Why this priority**: 指数增长误区是现代决策中常见且严重的问题，影响从技术发展预测到金融投资决策的各个领域。这是认知陷阱中最具代表性的线性思维错误。

**Independent Test**: 用户完成关于2^200规模和兔子繁殖问题的测试（10只兔子繁殖到80亿只），系统提供正确答案与用户猜测的对比，让用户意识到自己对指数增长的低估程度。

**Acceptance Scenarios**:

1. **Given** 用户进入指数增长测试页面, **When** 选择关于2的200次方的问题并提交答案, **Then** 系统显示正确答案（1.6×10^60）并解释其实际意义
2. **Given** 用户面临兔子繁殖问题, **When** 估算10只兔子每年翻5倍达到80亿所需时间, **Then** 系统显示实际计算结果（约11年）与用户的猜测进行对比
3. **Given** 用户完成所有指数增长相关测试, **When** 查看结果汇总, **Then** 系统展示用户预期与实际数值之间的巨大差异

### User Story 2 - 复利思维陷阱测试 (Priority: P2)

用户通过与银行贷款利息相关的选择题，体验复利思维陷阱，学习如何在长期决策中考虑复合效应。

**Why this priority**: 复利思维在财务规划、投资决策和债务管理中至关重要，但人们往往用线性思维来预测复利结果，这会导致巨大的决策偏差。

**Independent Test**: 用户完成关于复利计算的选择题（如比较不同利率和期限的贷款方案），系统提供复利vs线性增长的对比分析。

**Acceptance Scenarios**:

1. **Given** 用户面临复利计算问题, **When** 选择认为更划算的贷款方案, **Then** 系统显示实际复利计算结果并解释复利效应
2. **Given** 用户完成复利相关测试, **When** 查看长期影响对比, **Then** 系统提供复利vs线性增长的可视化展示

### User Story 3 - 历史经典决策失败案例重现 (Priority: P3)

用户通过重现历史上经典事故中的决策过程，体验和理解系统性决策错误，例如挑战者号发射决策、泰坦尼克号航线选择等。

**Why this priority**: 通过重现真实历史案例，用户可以更深刻地理解认知偏差在重大决策中的影响，这些案例具有强烈的教育意义。

**Independent Test**: 用户面对历史决策场景的选择题，系统提供当时的真实决策过程与正确分析的对比。

**Acceptance Scenarios**:

1. **Given** 用户进入历史决策案例页面, **When** 面对挑战者号发射决策情境, **Then** 系统展示当时的决策因素和错误判断
2. **Given** 用户完成历史案例测试, **When** 查看分析报告, **Then** 系统提供决策过程中的认知偏差分析

### User Story 4 - 互动式思维陷阱游戏 (Priority: P4)

用户通过游戏化的方式体验认知陷阱，比如类似狼人杀的推理游戏，让用户在互动中暴露各种思维局限。

**Why this priority**: 游戏化学习能够提高用户参与度和记忆效果，互动体验比单纯的选择题更能暴露用户的思维模式。

**Independent Test**: 用户参与一个完整的推理游戏，系统记录用户的推理过程和决策逻辑。

**Acceptance Scenarios**:

1. **Given** 用户开始推理游戏, **When** 进行推理和决策, **Then** 系统记录决策过程并在游戏结束后提供分析
2. **Given** 用户完成游戏, **When** 查看分析报告, **Then** 系统展示用户的认知偏差模式

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供指数增长误区测试，包括2^200规模、米粒问题、兔子繁殖问题（10只兔子约11年达到80亿只）等选择题
- **FR-002**: 系统 MUST 提供复利思维陷阱测试，包含与银行贷款利息相关的比较题
- **FR-003**: 系统 MUST 包含历史经典决策失败案例的重现，如挑战者号、泰坦尼克号等
- **FR-004**: 系统 MUST 提供互动式思维陷阱游戏，如推理游戏来暴露思维局限
- **FR-005**: 系统 MUST 记录用户对问题的预估答案与正确答案的对比，展示认知偏差程度
- **FR-006**: 系统 MUST 提供个人化反馈，指出用户在各种认知陷阱中的具体表现
- **FR-007**: 系统 MUST 提供结果汇总页面，展示用户预期与实际数值之间的差异
- **FR-008**: 系统 MUST 包含教育性说明，解释各种认知偏差的原理和影响

### Key Entities

- **挑战问题**: 包含问题ID、挑战类型、题目内容、具体化选项（如足球场数量、兔子繁殖时间选项）、正确答案、解释等属性
- **用户响应记录**: 包含用户ID、问题ID、用户选择、用户预估值、实际值、回答时间等属性
- **挑战结果汇总**: 包含用户ID、各类挑战得分、预估误差统计、改进建议、金字塔原理解释等属性
- **历史场景**: 包含场景ID、背景描述、决策选项、历史结果、分析说明等属性
- **解释框架**: 包含核心结论、支持论据、实例说明、行动建议等金字塔结构属性
